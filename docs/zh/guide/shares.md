# 分享链接

分享链接让你能将单个待办分组暴露给**团队外部**的人——客户、协作者或公众——而无需为他们提供 Prism 账号或团队成员身份。每个链接都是一个不可猜测的 URL，拥有自己的一组能力，因此你可以按需发布只读的路线图，或开放一个可协作的清单。

分享链接还能生成**可嵌入的 SVG 图片**（进度徽章和渲染的清单），你可以将其放入 README、Wiki 或仪表盘中。

---

## 创建分享链接

1. 打开你想分享的分组。
2. 打开**管理链接**（从分组的 `...` 菜单进入）。
3. 点击**新建链接**，为其命名，并选择其能力。
4. 复制生成的 URL 并分享。

创建、编辑和删除链接需要 `manage_set_links` 权限（默认授予所有者、联合所有者和管理员）。

---

## 能力

每个链接授予一组精确的能力。默认情况下，新链接为**仅查看**。

| 能力 | 允许的操作 |
| --- | --- |
| **查看** | 读取分组及其待办事项（默认 `开启`）。 |
| **创建** | 添加新的待办事项和子待办。 |
| **编辑** | 更改待办事项标题。 |
| **完成** | 切换待办事项的完成状态。 |
| **删除** | 删除待办事项。 |
| **评论** | 为未来的公开评论功能保留。 |
| **重排序** | 拖拽重新排序待办事项。 |

访问者只能看到并执行链接允许的操作；其他一切都在服务端被阻止。

::: warning
任何持有链接 URL 的人都可以在其授予的能力范围内使用它。请将启用写权限的链接视为密码，在不再需要时将其删除——删除链接会立即撤销访问。
:::

---

## 按邮箱限制

可以通过逗号分隔的**邮箱允许列表**将链接限制给特定的人。设置后：

- 访问者在分组加载前会被要求输入邮箱。
- 只有列表中的邮箱才被授予访问权限；其他人都会被拒绝。

将允许列表留空即为完全公开的链接。

::: info
邮箱检查是一道轻量级关卡，而非经过认证的身份验证——没有邮箱确认步骤。请用它来保持随手分享的链接私密，而非用于保护高度敏感的数据。
:::

## 可嵌入的徽章与清单

每个链接都可以渲染两张缓存的只读 SVG 图片。它们忽略能力标志和邮箱限制，因此任何持有令牌的人都能始终渲染。

### 进度徽章

一个 shields.io 风格的徽章，显示已完成/总数：

```markdown
![todos](https://glint.example.com/api/shared/<token>/badge.svg)
```

可通过查询参数自定义，如 `style=flat-square`、`label=Roadmap` 或 `color=blue`。

### 清单图片

分组的完整渲染清单：

```markdown
![checklist](https://glint.example.com/api/shared/<token>/todo-list.svg?theme=dark)
```

实用选项：`theme=dark`、`maxItems=10`、`width=480`、`showProgress=false`，以及颜色覆盖。完整参数列表参见[分享链接 API](../api/shares#embeddable-svgs)。

## 管理链接

在**管理链接**中你可以：

- 查看分组的每个链接，包括其能力和创建日期。
- 随时编辑链接的名称、能力或邮箱允许列表。
- 删除链接以立即撤销它。

团队级视图会列出所有分组的全部链接（并附带分组名称），便于审计。
