# 权限

Glint 拥有精细的、可配置的权限系统。团队所有者可以精确控制管理员和成员被允许执行的操作。

## 工作原理

- **所有者**始终拥有完全访问权限。这是硬编码的，不可被限制。
- **管理员**和**成员**角色拥有可配置的权限，并带有合理的默认值。
- 权限可以在**全局**（团队范围）或**按集合**（覆盖特定待办集合的全局设置）级别设置。

## 解析顺序

检查用户是否可以执行某项操作时：

1. 如果用户是**所有者**，始终允许。
2. 检查是否存在**按集合覆盖**（如果操作在特定集合内）。
3. 检查数据库中是否存在**全局覆盖**。
4. 回退到**内置默认值**。

## 权限键

| 键                    | 说明                              |
| --------------------- | --------------------------------- |
| `manage_settings`     | 编辑站点名称、Logo、品牌和应用配置  |
| `manage_permissions`  | 编辑权限规则（默认仅所有者）        |
| `manage_sets`         | 创建、重命名、删除和重新排序待办集合 |
| `create_todos`        | 创建新的待办事项                   |
| `edit_own_todos`      | 编辑用户自己创建的待办事项          |
| `edit_any_todo`       | 编辑他人创建的待办事项              |
| `delete_own_todos`    | 删除用户自己创建的待办事项          |
| `delete_any_todo`     | 删除他人创建的待办事项              |
| `complete_any_todo`   | 切换他人待办事项的完成状态          |
| `add_subtodos`        | 创建嵌套的子待办事项               |
| `reorder_todos`       | 拖拽重新排序待办事项               |
| `comment`             | 对待办事项添加评论                 |
| `delete_own_comments` | 删除用户自己发布的评论              |
| `delete_any_comment`  | 删除他人发布的评论                 |
| `view_todos`          | 查看集合中的待办事项               |

## 管理权限

1. 前往**设置 > 权限**（需要 `manage_permissions` 权限或所有者角色）。
2. 选择**范围**：全局（团队范围）或特定集合。
3. 为每个角色（管理员/成员）切换各项权限开关。
4. 点击**保存权限**。

使用**重置为默认值**可移除所选范围内的所有覆盖。

::: warning
只有所有者才能将 `manage_permissions` 权限授予管理员。这可以防止权限提升。
:::
