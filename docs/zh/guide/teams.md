# 团队与角色

Glint 使用 Prism 团队进行访问控制。每个待办集合都属于一个团队，你的角色决定了你的默认权限。

## 角色

共有三种角色，按权限等级从高到低排列：

- **所有者** — 拥有对所有内容的完全访问权限，包括设置和权限管理。不可被限制。
- **管理员** — 默认拥有广泛的访问权限，但权限可由所有者自定义。
- **成员** — 默认仅限于自己的条目，但权限可以扩展。

## 默认权限

| 权限             | 管理员 | 成员 |
| ---------------- | ------ | ---- |
| 管理设置          | 否     | 否   |
| 管理权限          | 否     | 否   |
| 管理集合          | 是     | 否   |
| 创建待办事项      | 是     | 是   |
| 编辑自己的待办    | 是     | 是   |
| 编辑任何待办      | 是     | 否   |
| 删除自己的待办    | 是     | 是   |
| 删除任何待办      | 是     | 否   |
| 完成任何待办      | 是     | 否   |
| 添加子待办        | 是     | 是   |
| 排序待办事项      | 是     | 否   |
| 评论             | 是     | 是   |
| 删除自己的评论    | 是     | 是   |
| 删除任何评论      | 是     | 否   |
| 查看待办事项      | 是     | 是   |

所有默认值均可全局或按集合覆盖。详见[权限](./permissions)。

## 团队切换

如果你属于多个团队，可使用侧边栏中的下拉菜单在团队之间切换。你的角色标识显示在侧边栏底部你名字的旁边。

## 管理团队

团队在你的 Prism 实例上进行管理。Glint 在登录时读取团队成员信息。团队成员变更将在下次登录时生效。

## 允许的团队 ID

如果在应用配置中设置了 `allowed_team_id`，则只有该特定 Prism 团队的成员才能登录 Glint。其他用户将看到 403 错误。
